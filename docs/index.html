How It Works:

  1. Season - Base temperature per season
  2. Biome - Biome-specific variations
  3. Time of Day - Warmer day, colder night
  4. Weather - Rain/storms reduce temperature
  5. Height (Y-level) - Temperature decreases with altitude
  6. Block Light - Light sources provide warmth
  7. Water Proximity - Water/ice reduce temperature
  8. Armor - Different armor provides warmth
  9. Food Level - Hunger affects temperature tolerance
  10. Movement - Sprinting generates body heat

Installation

1

Build the Plugin

Navigate to the plugin directory and compile using Maven:

cd BetterSeason
mvn clean package
2

Deploy JAR File

Copy the compiled JAR to your server's plugins directory:

cp target/BetterSeason-1.0.0.jar /path/to/server/plugins/
3

Start Your Server

The server will generate a default config on first run. Edit as needed and restart:

plugins/BetterSeason/config.yml
4

Verify Installation

Join your server and run the command to verify:

/season

Technical Details

Threading Model

  • Season Rotation: Every in-game day (24000 ticks)
  • Temperature Effects: Every second (20 ticks)
  • Particle Spawning: Every 2 seconds (40 ticks)
  • Asynchronous where possible

Data Persistence

  • Season data saved to config on change
  • Calendar state persisted between restarts
  • Temperature calculations are real-time
  • No database dependency

Compatibility

  • PlaceholderAPI: Ready for expansion
  • WorldGuard: Helper class support
  • Paper API: Latest features
  • Extensible: Custom modifiers

Performance

  • Efficient chunk-based processing
  • Particle effects limited to loaded chunks
  • Lightweight temperature calculations
  • Minimal memory footprint
